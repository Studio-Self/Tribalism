# Crisis Templates

Communication templates, post-mortem format, and crisis response checklists.

---

## Crisis Communication Templates

### Initial Acknowledgment (Within 4 Hours)
```
We want to address [situation] directly.

Here's what we know:
- [Fact 1]
- [Fact 2]
- [Fact 3 — or "We're still investigating [aspect]"]

What we've done so far:
- [Action 1]
- [Action 2]

What we're doing next:
- [Action 3]
- [Action 4]

We'll share a fuller update by [specific time — within 24 hours].

If you've been affected or have information to share:
→ [Contact method: DM, email, form]

All reports are confidential.

— [Name], [Role]
```

### Full Resolution Statement
```
## Update on [Situation] — [Date]

### What Happened
[Factual, chronological account. No speculation. 3-5 sentences.]

### What We Did
1. [Action taken and when]
2. [Action taken and when]
3. [Action taken and when]

### Impact
[Who was affected and how. Be specific but protect privacy.]

### What's Changing
To prevent this from happening again:
1. [Structural change 1]
2. [Structural change 2]
3. [Policy change]

### To Those Affected
[Direct, empathetic message to impacted members. Not corporate PR-speak.]

### Moving Forward
[How the community should proceed. What's expected of members.
Signal that the community is safe and active.]

Questions or concerns? [Contact method]

— [Name], [Role]
```

### Apology Template (When the Team Is at Fault)
```
I owe you all an apology.

[What happened — own it directly, no passive voice]

This was our mistake. Specifically, [what went wrong and why].

What we should have done: [what should have happened]

What we're doing now:
- [Corrective action 1]
- [Corrective action 2]
- [Preventive measure]

I'm sorry. Your trust is the foundation of this community, and
we take that seriously.

If you have questions or want to discuss this further, I'm available
[contact method].

— [Name]
```

## Post-Crisis Retrospective Template

```
## Crisis Retrospective — [Date]

### CONFIDENTIAL — Internal Use Only

### Incident Summary
- **What:** [Brief description]
- **When:** [Timeline — when discovered, when resolved]
- **Severity:** [Low / Medium / High / Critical]
- **Duration:** [How long the crisis lasted]
- **Members affected:** [Number and description]

### Timeline
| Time | Event |
|------|-------|
| [Time] | [Incident occurs / is reported] |
| [Time] | [First response] |
| [Time] | [Escalation] |
| [Time] | [Public communication] |
| [Time] | [Resolution] |
| [Time] | [Follow-up communication] |

### Root Cause Analysis
**Immediate cause:** [What directly caused the incident]
**Contributing factors:**
- [Factor 1]
- [Factor 2]
**Root cause:** [The underlying systemic issue]

### What We Did Well
- [Positive action 1]
- [Positive action 2]

### What We Could Improve
- [Improvement 1]
- [Improvement 2]

### Preventive Measures
| Action | Owner | Deadline | Status |
|--------|-------|----------|--------|
| [Action 1] | [Person] | [Date] | Not started |
| [Action 2] | [Person] | [Date] | Not started |
| [Action 3] | [Person] | [Date] | Not started |

### Community Impact Assessment
- **Member sentiment (post-crisis):** [Positive / Mixed / Negative]
- **Members lost:** [Number, if applicable]
- **Trust repair timeline:** [Estimated weeks/months]
- **Follow-up actions with affected members:** [Status]
```

## Crisis Severity Matrix

| Factor | Low | Medium | High | Critical |
|--------|-----|--------|------|----------|
| Safety risk | None | Potential | Active | Immediate |
| Members affected | <5 | 5-50 | 50-500 | 500+ |
| External visibility | Internal only | May spread | Spreading | National/viral |
| Trust impact | Minor | Moderate | Significant | Existential |
| Legal risk | None | Potential | Likely | Active |
| Response time | 24 hours | 4 hours | 1 hour | Immediate |

## Crisis Response Checklist

### First 30 Minutes
- [ ] Assess severity (use matrix above)
- [ ] Is there an ongoing safety risk? If yes, address immediately
- [ ] Remove harmful content if applicable
- [ ] Take immediate protective actions (mute, ban, lock down)
- [ ] Notify community team lead
- [ ] Assign roles: point person, communicator, investigator

### First 2 Hours
- [ ] Investigate: gather facts, screenshots, context
- [ ] Draft initial acknowledgment
- [ ] Get approval on communication (if company-affiliated)
- [ ] Post initial acknowledgment
- [ ] Set up private channel for affected members (if needed)

### First 24 Hours
- [ ] Complete investigation
- [ ] Make final decisions on consequences
- [ ] Draft resolution statement
- [ ] Contact affected members individually
- [ ] Post resolution statement
- [ ] Update community guidelines if needed

### First Week
- [ ] Monitor community sentiment
- [ ] Check in with affected members
- [ ] Implement immediate preventive measures
- [ ] Conduct internal retrospective
- [ ] Document lessons learned

### First Month
- [ ] Implement structural changes from retrospective
- [ ] Assess trust recovery
- [ ] Follow up with affected members
- [ ] Review and update crisis response procedures
